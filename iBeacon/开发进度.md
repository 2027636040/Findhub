# Google Find Hub Network (FMDN) — 设备端固件开发进度

## 项目目标

基于 SiFli-SDK 开发符合 Google Find Hub Network(FHN/FMDN v1.3)规范的防丢定位标签固件。

- 不实现 Fast Pair 自动配网弹窗;Account Key 由客户自研 App 下发
- 采用 160 位密钥旧版 BLE 方式(SECP160R1 曲线)
- 规范原文:`example/ble/iBeacon/README.md`(FHN v1.3)
- 参考实现:[GoogleFindMyTools](https://github.com/leonboe1/GoogleFindMyTools)(第三方,非官方)

## 代码结构

| 文件 | 职责 |
|------|------|
| `src/main.c` | FMDN 核心:广播 / EID 引擎 / GATT 挑战-应答 / 主流程 |
| `src/fmdn_auth.c/.h` | 认证模块:HMAC-SHA256 / AuthKey 计算校验 / 子密钥派生 / 一次性 nonce |
| `src/fmdn_store.c/.h` | 持久化模块:FlashDB KVDB(FAL 分区 `fmdn`),存 AK/EIK/配网标志/时钟 |
| `src/fmdn_rng.c/.h` | 硬件 TRNG 封装(nonce 随机源,bf0_hal_rng;不可用时回退 rand()) |
| `src/fmdn_flags.c/.h` | 广播帧 Hashed Flags 计算(电量/UT 位 xor SHA256(r)) |
| `src/fmdn_test_cmds.c` | 串口测试调试命令(`cmd_fmdn_*`) |
| `src/fmdn.h` | 共享接口:常量 / Data ID / app_env_t |
| `tools/verify_hashed_flags.py` | Hashed Flags / EID 的 Python 交叉验证脚本(零依赖,内置 AES) |
| `customer/boards/sf32lb52-core_n16r16/ptab.json` + `hcpu/custom_mem_map.h` | 新增 `fmdn` KV 分区(16KB) |

> 注:本板为 NOR,FlashDB 用 FAL 分区模式(`CONFIG_PKG_FDB_USING_FAL_MODE=y`),不依赖文件系统。

### GATT 挑战-应答协议(核心)
| 规范点 | 状态 | 备注 |
|--------|:----:|------|
| Beacon Actions 特征 `FE2C1238-...` 读/写/通知 | ✅ | |
| 一次性 nonce(每次读不同、用后失效、失败也失效) | ✅ | `fmdn_auth` nonce 生命周期 |
| 请求认证 HMAC-SHA256(各操作对应密钥) | ✅ | 0x00-0x08 全覆盖 |
| 响应(notify)认证段(消息尾部 0x01) | ✅ | 实测解密/比对通过 |
| 错误码 0x80 / 0x81 / 0x82 | ✅ | |
| 子密钥派生 Recovery/Ring/UT = SHA256(EIK‖0x01/02/03)[:8] | ✅ | 与 Python 一致 |
| 0x00 读参数:16B 数据 AES-ECB-128 加密 | ✅ | 实测解密=明文参数 |
| 0x01 读配网状态:state 位图 + 当前 EID | ✅ | bit0(EIK set)+ bit1(匹配 owner key);单 key 模型 |
| 0x02 SET_EIK:解密+持久化+断开后生效 | ✅ | 换钥验 EIK hash;无 hash 时校验"未配网才允许" |
| 0x03 CLEAR_EIK:清密钥+停广播+清持久化 | ✅ | 验 EIK hash;走完整工厂复位(清 account key) |
| 0x04 READ_EIK:recovery key + pairing mode + 0x82 + 加密 EIK | ✅ | 实测解密=原始 EIK |
| 0x05 Ring / 0x06 读响铃状态 | 🟡 | 认证 OK,但**无真实发声**,返回 mock |
| 0x07/0x08 UT 模式:帧类型 0x41/0x40 切换 | 🟡 | 0x08 已验 EIK hash;UT 开时已带 hashed flags;未做 MAC 轮换 / skip-ring-auth |
| Precision Finding(0x0A-0x0D) | ❌ | 可选 |
| Owner account key 概念(多账户槽 + owner key) | ❌ | 当前仅单个 account key |

## 总体结论

- **核心协议主线(加密 + GATT 挑战-应答 + EID + 配网 + 持久化)= 基本完成且实测通过。**
- **完整可认证产品 = 未达标**,主要缺口:DULT 防追踪服务、真实响铃硬件、地址隐私(RPA 轮换 + 去固定名)、message stream 授时、固件版本特征、配对/工厂复位策略。
- 其中 **DULT、响铃、RPA、不暴露标识** 为认证强制项,缺失则无法通过 Google 认证。
- D 组 EID 比对参照的是第三方实现 GoogleFindMyTools,**最终 100% 符合性需设备真机注册到 Google 后端到端验证**。

## 下一阶段建议优先级

1. RPA 地址轮换 + 去掉固定设备名(隐私强制,且影响入网)—— 单独专项
2. DULT Non-Owner 服务(认证强制,工作量大;依赖 IETF draft 文档)
3. 真实响铃(蜂鸣器驱动 + Ring/UT 联动)—— 依赖硬件
4. Message stream 授时 + 上电 Non-Discoverable FP(解决野外授时)
5. 完整多 account key 槽位 / 拒绝普通配对 / Precision Finding 等剩余项

> 已完成的小项:各项 EIK hash 校验(0x02/0x03/0x08)、设备主动 MTU 协商、硬件 TRNG nonce、
> Hashed Flags(UT 开时)、0x01 owner key 比特位、完整工厂复位(5 分钟保护 + CLEAR_EIK 清 AK)。
>
> **下次继续:优先 RPA 专项。** 建议先做 requirements spec,锁定上次"切 RPA 后手机反复自动
> 连/断"的根因,以及 near-owner/separated、UT 模式、1024s 同步轮换等边界,再进设计与实现。

## 快速开发参考

### 编译(纯 app)
```powershell
cd D:\SiFli-SDK\example\ble\iBeacon\project
scons --board=sf32lb52-core_n16r16_hcpu -j8
```
> 改了 `ptab.json` / `custom_mem_map.h`(分区表)时需**全量烧录**;仅改 src 代码烧 app 即可。

### 串口命令速查
```text
cmd_fmdn_ak <32hex>        设置 Account Key(并启动 5 分钟未配网保护)
cmd_fmdn_eik <64hex>       直接设 EIK(测试捷径,会落盘)
cmd_fmdn_stat              查看状态(配网/AK/EIK)
cmd_fmdn_time [unix]       设/查 Unix 时钟(设时落盘)
cmd_fmdn_subkeys           打印 Recovery/Ring/UT 子密钥
cmd_fmdn_eikhash <nonce16hex>   计算 SHA256(EIK||nonce)[:8](0x02换钥/0x03/0x08 用)
cmd_fmdn_rekeycmd [newEIK64hex] <nonce16hex>   生成 0x02 换钥写命令(nohash <nonce> 测无hash拒绝)
cmd_fmdn_authcmd <id> <nonce16hex> [add]   生成带正确 AuthKey 的写命令
cmd_fmdn_pairing on|off    切换配对模式(0x04 READ_EIK 需要)
cmd_fmdn_factory           立即工厂复位(清 account key + EIK + 持久化)
cmd_fmdn_eid_test / cmd_fmdn_debug / cmd_fmdn_aes / cmd_fmdn_gen   自测/调试
cmd_diss adv_start|adv_stop   广播启停
```

### 测试要点
- 通知 > 20 字节需在 nRF Connect 里 **Request MTU = 247**,否则被默认 MTU(23)截断。
- 一次性 nonce:每次 Write 前重新 Read 取新 nonce;用过即失效,**失败也作废**(否则报 `no valid nonce`)。
- UT 模式(0x07/0x08)切换在连接期间**不会**刷新广播,需**断开连接后**才生效(帧类型 0x40/0x41、hashed flags)。
- EID / Hashed Flags 交叉验证:`python tools/verify_hashed_flags.py <EIK64hex> <unix_ts> [ut]`;设备用 `cmd_fmdn_time <unix_ts>` 设相同时间戳后比对 EID 与 hashed flags 字节。
